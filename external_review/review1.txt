## Correction majeure apportée au script

**Problème détecté** : L'index FTS5 avec `content=` et `content_rowid=` provoquait une corruption SQLite (`database disk image is malformed`) lors de la synchronisation.

**Solution** : Table FTS5 **autonome** sans `content=` — synchronisation manuelle explicite via `sync_fts5()` :

```python
# AVANT (corrompu)
CREATE VIRTUAL TABLE search_specialites USING fts5(
    ..., content=specialites, content_rowid=rowid
)

# APRÈS (corrigé)
CREATE VIRTUAL TABLE search_specialites USING fts5(
    denomination, forme_pharmaceutique, voies_administration
)
```
