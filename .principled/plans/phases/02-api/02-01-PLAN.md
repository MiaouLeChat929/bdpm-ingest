# 02-01 — FTS5 Full-Text Search + axum API Scaffold

## Objective
Add full-text search to SQLite via FTS5, scaffold the axum HTTP API server, and implement `GET /drugs?q=` endpoint with FTS5 search.

## Design Decision: axum + tokio

Phase 1 used synchronous stack (ureq + rusqlite). Phase 2 adds async API:
- axum handles routing, extractors, middleware
- tokio runtime for async I/O
- `rusqlite` calls wrapped in `tokio::task::spawn_blocking` — rusqlite is blocking, spawn_blocking prevents blocking the async runtime
- Phase 1 import code unchanged (stays sync)

Files: `src/db/fts.rs`
Action:
```rust
/// Create FTS5 virtual table for drug search
pub fn create_fts_tables(conn: &Connection) -> Result<()> {
    conn.execute_batch(r#"
        CREATE VIRTUAL TABLE IF NOT EXISTS drugs_fts USING fts5(
            cis,
            name,
            form,
            lab_name,
            content='drugs',
            content_rowid='rowid'
        );

        -- Populate from drugs table
        INSERT INTO drugs_fts(rowid, cis, name, form, lab_name)
        SELECT rowid, cis, name, form, lab_name FROM drugs;

        -- Keep FTS in sync via triggers
        CREATE TRIGGER IF NOT EXISTS drugs_ai AFTER INSERT ON drugs BEGIN
            INSERT INTO drugs_fts(rowid, cis, name, form, lab_name)
            VALUES (new.rowid, new.cis, new.name, new.form, new.lab_name);
        END;

        CREATE TRIGGER IF NOT EXISTS drugs_ad AFTER DELETE ON drugs BEGIN
            INSERT INTO drugs_fts(drugs_fts, rowid, cis, name, form, lab_name)
            VALUES ('delete', old.rowid, old.cis, old.name, old.form, old.lab_name);
        END;

        CREATE TRIGGER IF NOT EXISTS drugs_au AFTER UPDATE ON drugs BEGIN
            INSERT INTO drugs_fts(drugs_fts, rowid, cis, name, form, lab_name)
            VALUES ('delete', old.rowid, old.cis, old.name, old.form, old.lab_name);
            INSERT INTO drugs_fts(rowid, cis, name, form, lab_name)
            VALUES (new.rowid, new.cis, new.name, new.form, new.lab_name);
        END;
    "#)?;
    Ok(())
}
```
Note: `content='drugs'` uses external content FTS5. This is correct — the FTS table references the actual drugs table. Without this, the drugs table would need to be shadowed (anti-pattern).
Verify: FTS query returns results

    let state = AppState {

```rust
// src/api/search.rs
pub async fn search_drugs(
    Query(params): Query<SearchParams>,
    State(state): State<AppState>,
) -> impl IntoResponse {
    let q = params.q.trim();
    if q.is_empty() {
        return Json(vec![]);
    }

    // spawn_blocking: rusqlite is blocking, must not run on async thread
    let results = tokio::task::spawn_blocking(move || {
        let conn = Connection::open(&state.db_path)?;
        let mut stmt = conn.prepare(
            "SELECT cis, name, form, lab_name
             FROM drugs_fts
             WHERE drugs_fts MATCH ?1
             LIMIT 20"
        )?;
        let rows = stmt.query_map([q], |row| Ok(DrugSearchResult {
            cis: row.get(0)?,
            name: row.get(1)?,
            form: row.get(2)?,
            lab_name: row.get(3)?,
        }))?.filter_map(|r| r.ok()).collect::<Vec<_>>();
        Ok::<_, rusqlite::Error>(rows)
    }).await??;

    Json(results)
}
```
