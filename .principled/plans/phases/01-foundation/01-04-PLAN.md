# 01-04 — Normalization Pipeline

## Objective
Implement all data normalizers: price (European comma + thousands separator → cents), date (DD/MM/YYYY, YYYYMMDD, and YYYY-MM-DD → ISO-8601), whitespace stripping, CIP stripping, generic type enum, smart apostrophe normalization, avis HTML stripping. Each normalizer is a pure function, tested independently by the tests defined in 01-03.

> **Note on normalization pipeline order:** Normalization is a separate pass from parsing — NOT embedded in the parser. This trades an extra data traversal for full testability of pure functions. Smart apostrophe normalization runs first (after encoding decode), before price/date/field normalization.

### Task 0: Smart apostrophe normalization (U+2019 → U+0027)
Files: `src/normalize/quotes.rs`

> **Why first:** Windows-1252 decode converts byte 0x92 → U+2019 (right single quotation mark). This normalization step runs immediately after decode, before any other normalization. It applies to ALL text fields — not just HAS files, since curly quotes can appear in any Windows-1252 encoded file.

Action:
```rust
/// Normalize curly apostrophe (U+2019) and left curly quote (U+2018) to straight apostrophe (U+0027).
/// 52,000+ occurrences across CIS_HAS_SMR (22,253) and CIS_HAS_ASMR (29,704).
/// Runs after Windows-1252 decode, before all other normalization.
pub fn normalize_apostrophe(text: &str) -> String {
    text.replace('\u{2019}', '\'')
         .replace('\u{2018}', '\'')
}

/// Also handle the Windows-1252 right single quotation mark (0x92 → U+2019 after decode).
/// After decode, the text may contain U+2019 which this function normalizes.
```

- `"l'indication"` (U+2019) → `"l'indication"` ✓

### Task 1: Price normalization — cents with thousands separator handling
Files: `src/normalize/price.rs`
Action:
```rust
/// Parse a European-format price string to integer cents.
/// Handles: "24,34" → 2434, "1,466,29" → 146629, "" → None
pub fn parse_price_cents(raw: &str) -> Result<Option<i64>> {
    let trimmed = raw.trim();
    if trimmed.is_empty() { return Ok(None); }

    // Count commas — if 2+, first commas are thousands separators
    let comma_count = trimmed.chars().filter(|&c| c == ',').count;
    match comma_count {
        0 => {
            // "24" → 2400 cents (integer euros)
            let euros: f64 = trimmed.parse()?;
            Ok(Some((euros * 100.0).round() as i64))
        }
        1 => {
            // "24,34" → 2434 cents
            let normalized = trimmed.replace(',', ".");
            let val: f64 = normalized.parse()?;
            Ok(Some((val * 100.0).round() as i64))
        }
        2.. => {
            // "1,466,29" → remove ALL commas → "146629" → parse → 146629
            // Wait — that's wrong. "1,466,29" = 1,466.29 euros.
            // Correct: remove thousands-separator commas (all except last),
            // then replace last comma with dot.
            // Simpler: split on commas. If last segment has exactly 2 digits,
            // that's the decimal part. Everything else is thousands groups.
            let parts: Vec<&str> = trimmed.split(',').collect();
            let last = parts.last().unwrap_or(&"");
            if last.len() == 2 {
                // Decimal part: join integer parts, add decimal
                let integer_part: String = parts[..parts.len()-1].join("");
                let full = format!("{}.{}", integer_part, last);
                let val: f64 = full.parse()?;
                Ok(Some((val * 100.0).round() as i64))
            } else {
                // All commas are thousands separators, no decimal
                let full: String = trimmed.replace(',', "");
                let val: f64 = full.parse()?;
                Ok(Some((val * 100.0).round() as i64))
            }
        }
    }
}
```
Edge cases tested:
- `"24,34"` → `2434`
- `"1,466,29"` → `146629`
- `""` → `None`
- `"0"` → `0`
Verify: All `tests/normalization.rs` price tests pass
Done: `parse_price_cents` handles all documented price formats

            // Used by CIS_InfoImportantes.txt

        }
    }
}

### Task 3: Whitespace + CIP + encoding normalization
Files: `src/normalize/fields.rs`
Action:
```rust
/// Strip leading/trailing whitespace from a field.
/// Specifically handles the BDPM quirk where lab names start with ' '.
pub fn strip_field(raw: &str) -> String {
    raw.trim().to_string()
}

/// Normalize double-spaces in a string (CIS_GENER names).
pub fn normalize_spaces(raw: &str) -> String {
    raw.split_whitespace().collect::<Vec<_>>().join(" ")
}

/// Strip 34009 prefix from EAN to get canonical CIP-7.
/// If the raw value is already 7 digits, return as-is.
/// If it starts with "34009" and is 13 digits, return last 7.
pub fn strip_cip_ean(raw: &str) -> String {
    if raw.len() == 13 && raw.starts_with("34009") {
        raw[6..].to_string() // last 7 digits
    } else if raw.len() == 7 && raw.chars().all(|c| c.is_ascii_digit()) {
        raw.to_string()
    } else {
        raw.to_string() // pass through unchanged
    }
}

/// Normalize reimbursement rate: "65%" or "65 %" → 0.65
pub fn normalize_reimb_rate(raw: &str) -> Option<f32> {
    let cleaned = raw.trim().replace(' ', "").trim_end_matches('%');
    cleaned.parse::<f32>().ok().map(|v| v / 100.0)
}

/// Normalize generic type: "0"→"reference", "1"→"generic", "2"→"cross-group", "4"→"sustained-release"
pub fn normalize_generic_type(raw: &str) -> &'static str {
    match raw.trim() {
        "0" => "reference",
        "1" => "generic",
        "2" => "cross-group",
        "4" => "sustained-release",
        _ => "unknown",
    }
}

/// Boolean normalization: "Oui"→true, "Non"→false
pub fn normalize_boolean(raw: &str) -> bool {
    matches!(raw.trim(), "Oui" | "oui" | "OUI")
}
```

### Task 4: avis HTML stripping
Files: `src/normalize/html.rs`
Action:
```rust
/// Strip HTML tags from avis field text.
/// Replaces <br> with newline, strips <p>, <b>, and other tags.
/// Preserves text content.
pub fn strip_avis_html(raw: &str) -> String {
    let result = raw
        .replace("<br>", "\n")
        .replace("<br/>", "\n")
        .replace("<br />", "\n")
        .replace("<BR>", "\n")
        .replace("</p>", "\n")
        .replace("</P>", "\n");
    // Remove remaining HTML tags
    let re = regex_lite::Regex::new("<[^>]*>").unwrap(); // or manual scan
    re.replace_all(&result, "").to_string()
}
```
Add `regex-lite = "0.1"` to Cargo.toml for tag stripping (lighter than full `regex` crate).

### Task 5: Deduplication helper for CIS_COMPO

/// Remove exact duplicate rows from CIS_COMPO (same CIS+substance_code+dosage).
/// 4,780 duplicates detected (32,389 total − 27,609 unique).
/// Uses HashSet<(String,String,String)> to deduplicate.
pub fn dedup_compo(rows: Vec<ValidatedRow>) -> Vec<ValidatedRow> {
    let mut seen = HashSet::new();
    rows.into_iter().filter(|r| {
        if r.fields.len() < 5 { return true; } // keep malformed for logging
        let key = (r.fields[0].clone(), r.fields[2].clone(), r.fields[4].clone());
        seen.insert(key)
    }).collect()
}
