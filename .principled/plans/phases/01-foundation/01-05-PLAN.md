# 01-05 — Database Init + Schema Migration + Full Import

## Objective
Create the SQLite database with all tables from BRIEF.md schema. Implement the import pipeline: validated + normalized rows → typed INSERT into each table. The import runs in a transaction per file (all-or-nothing for that file). On success, import_log records the BLAKE3 hash, row count, and timestamp. On failure, transaction rolls back and previous state is preserved.

### Task 1: Database initialization + migration framework
Files: `src/db/mod.rs`, `src/db/migrations.rs`
Action:
- `fn init_db(path: &Path) -> Result<Connection>`:
  1. Open SQLite connection with `rusqlite::Connection::open(path)`
 2. Enable WAL mode: `PRAGMA journal_mode=WAL`
 3. Set synchronous: `PRAGMA synchronous=NORMAL`
 4. Enable FK enforcement (for validation only — orphan tables handled separately):
- **Bulk-insert optimization**: during import, temporarily set `PRAGMA synchronous=OFF` and `PRAGMA cache_size=-64000` for maximum write throughput. Restore to `PRAGMA synchronous=NORMAL` and `PRAGMA cache_size=-2000` after COMMIT:
  ```rust
  fn optimize_for_bulk_insert(conn: \&Connection) {
      conn.execute_batch("PRAGMA journal_mode=WAL; PRAGMA synchronous=OFF; PRAGMA cache_size=-64000; PRAGMA temp_store=MEMORY;").unwrap();
  }
  fn restore_normal_settings(conn: \&Connection) {
      conn.execute_batch("PRAGMA synchronous=NORMAL; PRAGMA cache_size=-2000;").unwrap();
  }
  ```
    `PRAGMA foreign_keys=ON`

- Use `rusqlite_migration` crate for versioned migrations:

### Task 3: Master import orchestrator
Files: `src/import/mod.rs`
Action:
```rust
pub fn import_all(conn: &Connection, raw_dir: &Path, state: &StateStore) -> Result<ImportReport> {
    // Import order respects FK: drugs first (upsert), then all others
    // Orphan tables (SMR/ASMR/GENER): disable FK during their import phase
    // COMPO: 0 orphans — FK enforcement OK
    // mitm/atc_codes: FK enforced for mitm→drugs (0 orphans in current file)
    let order = [
        BDPMFile::CIS_bdpm,           // master drug records
        BDPMFile::CIS_CIP,            // FK → drugs
        BDPMFile::CIS_COMPO,          // FK → drugs
        BDPMFile::CIS_CPD,            // FK → drugs
        BDPMFile::CIS_GENER,          // FK → drugs (informational, orphans allowed)
        BDPMFile::CIS_MITM,           // mitm FK → drugs; atc_codes has no FK to drugs
        BDPMFile::CIS_HAS_SMR,        // FK → drugs (orphans allowed)
        BDPMFile::CIS_HAS_ASMR,       // FK → drugs (orphans allowed)
        BDPMFile::HAS_LiensPageCT,    // FK → drugs (via ct_id)
        BDPMFile::CIS_CIP_Dispo,      // FK → drugs (via CIS)
    ];

    let mut report = ImportReport::default();
    for file in &order {
        let path = raw_dir.join(file.filename());
        let schema = file.schema();
        let (rows, validation) = validate_file(&path, schema)?;

        // Normalize
        let normalized = normalize_rows(file, rows)?;

        // Import
        match import_file(conn, file, &normalized) {
            Ok(stats) => {
                state.mark_updated(file, &hash, size);
                report.success(file, stats);
            }
            Err(e) => {
                report.failure(file, e);
                // Continue with other files (partial import acceptable)
            }
        }
    }
    state.save(&state_path)?;
    Ok(report)
}
```
CLI: `bdpm-ingest import [--full] [--file=NAME]`
- `--full`: re-import all files regardless of state
- `--file=NAME`: import only one file
- `--data-dir <PATH>`: override data directory (for CI runners)
- Default: import only files whose BLAKE3 hash differs from state
- **CI artifact design**: The output SQLite file lives at `{data_dir}/bdpm.db`. After import, run `sqlite3 {data_dir}/bdpm.db "PRAGMA wal_checkpoint(TRUNCATE);"` to produce a single portable `.db` for GitHub Release upload.
Verify: `bdpm-ingest import` creates all tables and imports all 10 files
Done: Full import works end-to-end
