## Tasks

### Task 1: OpenAPI spec with utoipa
Files: `src/api/openapi.rs`, `openapi.yaml`
Action:
```rust
use utoipa::OpenApi;

#[derive(OpenApi)]
#[openapi(
    paths(api::search::search_drugs, api::drugs::drug_detail),
    components(schemas DrugSearchResult, DrugDetail, Presentation, Composition),
    tags("BDPM Drug API")
)]
pub struct ApiDoc;

#[utoipa::path(get, path = "/openapi.yaml", responses(
    (200, description = "OpenAPI spec", content_type = "application/x-yaml")
))]
pub async fn openapi_yaml() -> impl IntoResponse {
    let yaml = ApiDoc::openapi().to_yaml().unwrap();
    ([(header::CONTENT_TYPE, "application/x-yaml")], yaml)
}
```

Run `cargo build` and extract spec:
```bash
cargo run -- serve --dump-openapi > openapi.yaml
```
Verify: `openapi.yaml` is valid OpenAPI 3.x
Done: OpenAPI spec generated

### Task 2: Operational runbook
Files: `docs/runbook.md`
Action:
```markdown
# BDPM Operational Runbook

## Monitoring

### Import Failures
- Alert: `import_log` has `status = 'failed'` rows
- Response: Check `import_log.bad_rows` count, compare to previous run

### Row Count Deviation
- Alert threshold: <90% of previous `row_count` for any file
- Query:
  ```sql
  SELECT file_name, row_count,
         LAG(row_count) OVER (ORDER BY imported_at) as prev,
         ROUND(100.0 * row_count / prev) as pct
  FROM import_log
  WHERE status = 'success'
  ORDER BY imported_at DESC LIMIT 10;
  ```

### Schema Drift (CI Failure)
- CI fails if field count changes
- Response: Audit BDPM ANSM format documentation, update FileSchema and tests

## Manual Operations

### Force Full Reimport
```bash
./target/release/bdpm-ingest import --full --data-dir data
```

### Single File Reimport
```bash
./target/release/bdpm-ingest import --file CIS_CIP_Dispo_Spec.txt --data-dir data
```

### Check Current State
```bash
./target/release/bdpm-ingest stats
./target/release/bdpm-ingest logs --limit 5
```

## Schema Change Response

When BDPM changes a file's field count or format:

1. **Detect**: CI fails on `field_count` assertion
2. **Audit**: Download new file, compare to previous
3. **Update**: Modify `FileSchema` in `src/download/manifest.rs`
4. **Test**: Update integration tests in `tests/`
5. **Migrate**: Add migration if schema changes affect table structure
6. **Deploy**: PR with updated schema + migration
7. **Monitor**: Watch import log for new `bad_rows` count
```
