# 03-01 — Sync Engine: Change Detection + Full-Table Reload

## Objective
Implement the monthly sync pipeline: file-level BLAKE3 change detection, per-file full-table truncate+reload, and the import log viewer.

## Context
@.principled/plans/BRIEF.md
@.principled/plans/ROADMAP.md
Requires: Phase 1 complete

## Design Decisions

**No row-level delta:** BDPM files have no timestamps. Delta is impossible. Full-table reload per changed file is correct.

**Orphan preservation:** Withdrawn drugs disappear from CIS_bdpm but SMR/ASMR/GENER reference them. INSERT OR REPLACE for drugs preserves historical references without orphan data loss.

**BLAKE3 for change detection:** Computed post-download. Content-Length is an optimization (skip GET if size matches stored).

## Tasks

### Task 1: Per-file change detection
Files: `src/sync/detector.rs`
Action:
```rust
pub struct SyncPlan {
    pub file: BDPMFile,
    pub reason: ChangeReason,
    pub hash: String,
    pub size: u64,
}

pub enum ChangeReason {
    FirstRun,
    SizeChanged,
    HashChanged,
}

pub fn detect_changes(
    state: &StateStore,
    raw_dir: &Path,
    fetcher: &Fetcher,
) -> Result<Vec<SyncPlan>> {
    let mut plans = Vec::new();

    for file in BDPMFile::all() {
        let url = format!("{}{}", BDPM_URL, file.download_path());
        let bytes = fetcher.fetch(&url, raw_dir)?;
        let hash = blake3::hash(&bytes).to_hex().to_string();
        let size = bytes.len() as u64;

        if state.needs_update(file, &hash, size) {
            let reason = match state.files.get(file.filename()) {
                None => ChangeReason::FirstRun,
                Some(prev) if prev.size_bytes != size => ChangeReason::SizeChanged,
                _ => ChangeReason::HashChanged,
            };
            plans.push(SyncPlan { file: file.clone(), reason, hash, size });
        }
    }
    Ok(plans)
}
```

### Task 2: Per-file truncate + reload
Files: `src/sync/pipeline.rs`
Action:
```rust
pub fn sync_file(
    conn: &Connection,
    plan: &SyncPlan,
    rows: &[NormalizedRow],
) -> Result<ImportStats> {
    let table = plan.file.target_table();

    // BEGIN IMMEDIATE: acquires write lock immediately or fails
    let tx = conn.immediate_transaction()?;

    // Truncate (preserve historical data for withdrawn drugs via INSERT OR REPLACE for drugs table)
    if plan.file == BDPMFile::CIS_bdpm {
        // drugs table: INSERT OR REPLACE (upsert) preserves SMR/ASMR/GENER references
        // No truncate needed — upsert updates or inserts
    } else {
        tx.execute(&format!("DELETE FROM {table}"), [])?;
    }

    // Bulk insert (prepared statement with transaction)
    let mut insert = tx.prepare_cached(&plan.file.insert_sql())?;
    for row in rows {
        insert.execute(params![...])?;
    }

    tx.commit()?;
    Ok(stats)
}
```
Note: drugs uses INSERT OR REPLACE — never truncated. All other tables are truncated before reload. This preserves SMR/ASMR/GENER references to withdrawn drugs.

```rust
pub fn view_import_log(conn: &Connection, limit: usize) -> Result<Vec<ImportLogEntry>> {
    let mut stmt = conn.prepare(
        "SELECT file_name, file_hash, row_count, status, bad_rows, skipped_rows, duration_ms, imported_at
         FROM import_log
         ORDER BY imported_at DESC
         LIMIT ?1"
    )?;
    let rows = stmt.query_map([limit], |row| Ok(ImportLogEntry {
        file_name: row.get(0)?,
        file_hash: row.get(1)?,
        row_count: row.get(2)?,
        status: row.get(3)?,
        bad_rows: row.get(4)?,
        skipped_rows: row.get(5)?,
        duration_ms: row.get(6)?,
        imported_at: row.get(7)?,
    }))?.filter_map(|r| r.ok()).collect();
    Ok(rows)
}
```
