# 01-01 — Project Scaffold + Download Fetcher + State Store

## Objective
Set up the Rust project structure, define the `BDPMFile` manifest with full per-file schema metadata, implement a polite HTTP fetcher, and build the state store that tracks BLAKE3 hashes for change detection. This plan uses a synchronous stack (no tokio) since SQLite is fundamentally synchronous and the CLI only downloads 10 files monthly. Content-Length is an optimization only; BLAKE3 hash is the authoritative signal.

## Context
@.principled/plans/BRIEF.md
@.principled/plans/ROADMAP.md

## Key Design Decisions

**State store (`import_state.json`):** Created on first run. On subsequent runs, compare stored BLAKE3 hash against newly computed hash after download. Content-Length from HTTP HEAD is consulted as a cheap pre-filter (skip the GET if HEAD size == stored size), but BLAKE3 after download is the only authority.

**First run:** No state exists → `needs_update` always returns true → always downloads. After first download, state store has BLAKE3 hash → subsequent runs compare hash.

**Rate limiting:** 5s between requests, 3 retries with exponential backoff (5s, 10s, 30s).

**Why this stack (not the original tokio + reqwest):**
- **No tokio**: SQLite is synchronous. rusqlite is synchronous. The CLI downloads 10 files monthly. Async adds ~60s compile time + ~4MB binary size for zero benefit.
- **ureq over reqwest**: ureq is a synchronous HTTP client. 1/5 the compile time of reqwest, no async runtime. reqwest::blocking still pulls in the full hyper/tokio stack.
- **blake3 over sha2**: Content identity for change detection, not cryptographic security. BLAKE3 is 4-10x faster, modern, parallel-capable.
- **encoding_rs**: BDPM files arrive as raw bytes in two encodings. `std::str::from_utf8` fails on Latin-1. This was missing from the original plan.

### Task 3: Implement polite HTTP fetcher
Files: `src/download/fetcher.rs`
Action:
- `Fetcher`: struct holding `user_agent: String`, `rate_limit: Duration`
- `Fetcher::fetch(&self, url: &str, dest_dir: &Path) -> Result<(Vec<u8>, u64, String)>`:
  1. Build request with `ureq::agent()` and `User-Agent: bdpm-ingest/0.1 (contact@example.com)`
  2. Set per-request timeout: 60 seconds (`ureq::Request::timeout()`)
  3. GET request (synchronous — no async runtime needed)
  4. Read response body into `Vec<u8>`
  5. Save bytes to `{dest_dir}/{filename}`
  6. Compute BLAKE3 hash: `blake3::hash(&bytes).to_hex().to_string()`
  7. Return `(bytes, content_length, blake3_hex)`
  8. On HTTP error or timeout: retry 3× with `std::thread::sleep` (5s, 10s, 30s backoff), cap backoff at 1 hour for server-wide outages
  9. Every request: `std::thread::sleep(Duration::from_secs(5))` rate limit before next
- No tokio, no async — pure synchronous I/O
Verify: `cargo build` succeeds

### Task 4: State store with BLAKE3 hash tracking
Files: `src/storage/state.rs`
Action:
```rust
#[derive(Serialize, Deserialize, Default)]
pub struct StateStore {
    pub files: HashMap<String, FileState>,
}

#[derive(Serialize, Deserialize)]
pub struct FileState {
    pub content_hash: String,  // BLAKE3 hash — authoritative change signal
    pub size_bytes: u64,
    pub downloaded_at: String, // ISO-8601
}

impl StateStore {
    pub fn load(path: &Path) -> Result<Self> { /* JSON load */ }
    pub fn save(&self, path: &Path) -> Result<()> { /* JSON save */ }
    pub fn needs_update(&self, file: &BDPMFile, new_hash: &str, new_size: u64) -> bool {
        // FIRST RUN: no state → always true
        // Subsequent runs: BLAKE3 hash differs → needs update
        match self.files.get(file.filename()) {
            None => true,  // first run
            Some(prev) => prev.content_hash != new_hash || prev.size_bytes != new_size,
        }
    }
    pub fn mark_updated(&mut self, file: &BDPMFile, hash: &str, size: u64) {
        self.files.insert(file.filename().to_string(), FileState {
            content_hash: hash.to_string(),
            size_bytes: size,
            downloaded_at: Utc::now().to_rfc3339(),
        });
    }
}
```
State file location: `{data_dir}/import_state.json`
Done: StateStore serializes/deserializes correctly
