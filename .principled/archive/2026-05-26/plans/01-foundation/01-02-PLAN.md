# 01-02 — Tab Parser with Per-File Encoding + Field-Count Guard

## Objective
Build a streaming tab-separated value parser that handles BDPM's specific quirks: per-file encoding (Latin-1 vs UTF-8), LF/CRLF/mixed line endings, Windows-1252 residual bytes, phantom empty fields, multi-line record detection, field-count validation against known schema, and malformed-row filtering. The parser must be safe to run against all 10 files and emit structured rows or clear errors — never silently corrupt data.

## Tasks

### Task 1: Encoding + tab parser with line-ending/quirk handling
Files: `src/parse/tab_parser.rs`
Action:
- `struct TabParser<R: BufRead>`: streaming parser over a `BufRead`
- `fn from_file(path: &Path, encoding: Encoding) -> Result<Self>`:
  - Open file as `File`, wrap in `BufReader`
  - Decode using `encoding_rs::ISO_8859_1` or `UTF_8` (use `encoding_rs::WINDOWS_1252` for 7 files; ISO-8859-1 does NOT handle 0x92 → U+2019)
  - Treat `encoding_rs::Decoder` for correct byte→char mapping
  - Split on `\t` (tab)
  - Handle line endings by stripping all trailing `\r` from each line

### Phantom field stripping
CIS_CIP_bdpm.txt has trailing tabs on 100% of lines (20,903/20,903), creating a phantom empty 14th field. Strip trailing empty fields after tab split:
```
let fields: Vec<&str> = line.split('\t').collect();
while fields.last() == Some(&"") { fields.pop(); }
```
Apply this to all files — it is harmless for files without trailing tabs.
Note: CIS_CIP_Dispo_Spec does NOT have trailing tabs — fields 2 and 7 are intentionally empty mid-record, creating double-tab patterns. Do NOT treat this as a phantom field.

### Multi-line record detection (CIS_HAS_SMR/CIS_HAS_ASMR)
Long avis fields may contain embedded newlines. Algorithm:
- If line starts with a 8-digit CIS code → new record
- Else → append to previous record's avis field (join with space)
- `fn next_record(&mut self) -> Result<Option<Record>>`:
  - Returns `Record` (Vec<String>) or error
  - Strips trailing `\r` from each line before split
  - Empty lines: skip
- `Record`: `pub fields: Vec<String>`, `pub line_number: usize`
- `Error` variants: `EncodingError`, `UnexpectedEOF`, `TooManyFields`, `IoError(std::io::Error)`
Verify: `cargo test` on encoding round-trips for all 10 files
Done: TabParser reads all 10 files without panic

### Task 2: Field-count guard + row iterator
Files: `src/parse/iter.rs`
Action:
- `fn parse_file(path: &Path, schema: &FileSchema) -> Result<RowIterator<'_>>`:
  - Creates `TabParser`, wraps in `RowIterator`
  - `RowIterator`: holds `TabParser`, `FileSchema`
  - `fn next(&mut self) -> Result<Option<ValidatedRow>>`:
    - Reads raw `Record`
    - Checks `record.fields.len() == schema.field_count`
    - If mismatch: **log to import_log with line_number, emit None, increment `skipped_rows` counter**
    - If match: emit `ValidatedRow { fields, line_number, warnings: Vec<String> }`
  - Warnings emitted for: empty fields where non-empty expected, suspicious field values
- `struct ValidatedRow { fields: Vec<String>, line_number: usize, warnings: Vec<String> }`
- Implement `Iterator<Item = Result<ValidatedRow>>` for `RowIterator`
- **Note: CIS_HAS_SMR/ASMR** — current snapshot has 0 malformed rows. Field-count guard is defensive design; if BDPM publishes a changed file with field_count ≠ 6, the guard filters it. No malformed rows expected.
Verify: Parser reads CIS_HAS_SMR_bdpm.txt and all rows have field_count = 6 (0 malformed in current snapshot)
Done: Field-count guard fires on CIS_HAS_SMR rows with wrong field count

### Task 4: Quirk detection + warnings
Files: `src/parse/quirks.rs`

- `fn emit_warning(file: &BDPMFile, row: &ValidatedRow) -> Vec<String>`:
