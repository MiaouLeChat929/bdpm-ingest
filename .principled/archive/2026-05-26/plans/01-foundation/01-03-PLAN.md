# 01-03 — FileSchema Validation + CI Regression Tests

## Objective
Take the validated row iterator from 01-02 and wrap it with per-file schema validation. Define the invariant assertions (row counts, field counts, referential integrity, price format correctness) and embed them as compile-time constants and integration tests. This is the foundation for CI — when the BDPM publishes a file with a changed field count or unexpected row count, CI must fail before production re-import.

## Context
@.principled/plans/BRIEF.md
@.principled/plans/phases/01-foundation/01-02-PLAN.md

Requires: 01-02 complete (row iterator functional for all 10 files)

## Key Insight

The 01-02 parser emits rows or skips malformed ones. 01-03 wraps it with the **FileSchema gate**: actual field count vs. expected field count. The result is a `ValidatedIterator` that guarantees field uniformity. The integration tests use the same gate — when field count changes, CI fails.

### Task 5: Orphan CIS regression test
Files: `tests/orphan_tracking.rs`

// SMR/ASMR/GENER reference withdrawn drugs — orphan counts are EXPECTED.
// This test tracks the known orphan counts so schema changes are visible.
#[test]
fn smr_orphan_count_within_expected_range() {
    // Known baseline: 2,806 orphans (18.4% of SMR rows)
    // Track as upper bound — if it grows significantly, BDPM withdrew many drugs
    let orphans = query!("SELECT COUNT(*) FROM smr s LEFT JOIN drugs d ON s.cis = d.cis WHERE d.cis IS NULL").unwrap();
    assert!(orphans < 3500, "SMR orphan count {} exceeds expected range (baseline: 2806) — possible data anomaly", orphans);
}

#[test]
fn asmr_orphan_count_within_expected_range() {
    // Known baseline: 1,567 orphans (15.8% of ASMR rows)
    let orphans = query!("SELECT COUNT(*) FROM asmr a LEFT JOIN drugs d ON a.cis = d.cis WHERE d.cis IS NULL").unwrap();
    assert!(orphans < 2000, "ASMR orphan count {} exceeds expected range (baseline: 1567)", orphans);
}

#[test]
fn generic_orphan_count_within_expected_range() {
    // Known baseline: 2,503 orphans (23.5% of GENER rows — highest orphan rate)

}
