### Task 3: Fix dilution regex false positive on gene/cell therapy products
Files: src/normalize/mod.rs:30-32, 64
Action: The pattern `\b\d+\s*(?:CH|DH|K|X|LM)\b` matches "2 X" in "2 x 100 000 000 cellules" — the `X` is a multiplication operator, not a dilution unit. 14 gene/cell therapy products are incorrectly filtered (TECARTUS, YESCARTA, KYMRIAH, ZOLGENSMA, etc.).

Fix: Remove `X` from the alternation and add word-boundary enforcement. The valid homeopathic dilution patterns are `CH` (Hahnemannian centesimal), `DH` (decimal hahnemannian), `K` (Korsakovian), and `LM` (50-millesimal). Standalone `X` is never a dilution unit.

Change line 30-32 from:
```rust
Regex::new(r"(?i)\b\d+\s*(?:CH|DH|K|X|LM)\b").unwrap()
```
To:
```rust
Regex::new(r"(?i)\b\d+\s*(?:CH|DH|K|LM)\b").unwrap()
```

This preserves detection of `4CH`, `30DH`, `5K`, `15LM` while not matching `2 X 100 000 000`.

Add unit test in the tests section of normalize/mod.rs:
```rust
#[test]
fn test_dilution_regex_no_false_positive() {
    assert!(DILUTION_RE.is_match("4CH à 30CH"));
    assert!(DILUTION_RE.is_match("5K"));
    assert!(DILUTION_RE.is_match("15LM"));
    assert!(!DILUTION_RE.is_match("2 x 100 000 000 cellules")); // multiplication, not dilution
    assert!(!DILUTION_RE.is_match("1 X 10")); // multiplication
}
```
Verify: `cargo test dilution --lib` passes
Done: Gene/cell therapy products no longer filtered. Homeopathic dilution patterns still detected.

```rust
pub fn canonicalize_route(route: &str) -> String {
    let normalized = strip_diacritics(&route.trim().to_lowercase());
    // Handle semicolon-separated multi-value routes
    if normalized.contains(';') {
        return normalized.split(';')
            .map(|part| {
                let trimmed = part.trim();
                ROUTE_CANONICAL.get(trimmed).map(|s| s.to_string()).unwrap_or(trimmed.to_string())
            })
            .collect::<Vec<_>>()
            .join(";");
    }
    ROUTE_CANONICAL.get(&normalized).map(|s| s.to_string()).unwrap_or(normalized)
}
```
