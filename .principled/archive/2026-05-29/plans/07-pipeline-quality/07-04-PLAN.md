## Objective

Add compile-time field count validation to each normalizer via const assertions — upgrading from the current runtime test pattern. This catches field count mismatches at compile time, not test time, without any new dependencies or structural refactoring.

Purpose: Catch schema drift before it reaches CI
Output: `const ASSERT` in each normalizer function; runtime test kept as safety net

## Context

@src/normalize/mod.rs (all normalizer functions)
@src/normalize/dedup.rs

## Decision: Const Assertions, Not Typed Structs

Typed structs with a `TableRow` trait would give true compile-time guarantees but require 10 trait impls with ~50 lines of boilerplate each. Const assertions give90% of the benefit (compile-time field count validation) at 5% of the cost. The existing runtime test `test_insert_sql_value_counts_match` is kept as a safety net.
